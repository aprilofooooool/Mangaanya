# Mangaanya システム仕様書

## 概要

Mangaanyaは、漫画ファイル（ZIP/RAR形式）を管理するためのWPFアプリケーションです。ファイルのスキャン、メタデータの管理、AI による情報取得、サムネイル生成などの機能を提供します。

## 技術スタック

- **フレームワーク**: .NET 9.0 (Windows)
- **UI**: WPF (Windows Presentation Foundation)
- **データベース**: SQLite
- **依存性注入**: Microsoft.Extensions.DependencyInjection
- **MVVM**: CommunityToolkit.Mvvm
- **JSON**: Newtonsoft.Json
- **圧縮ファイル処理**: SharpCompress
- **AI API**: Google Gemini API

## 主要機能

### 1. ファイル管理機能

#### フォルダスキャン
- 指定フォルダ内の ZIP/RAR ファイルを検索
- サブフォルダを含むスキャンに対応
- 増分スキャンと全体スキャンの両方をサポート
- 進捗表示とキャンセル機能

#### ファイル情報管理
- ファイル名からのメタデータ自動抽出
- 手動でのメタデータ編集
- ファイルサイズ、作成日時、更新日時の管理

### 2. メタデータ管理

#### 対応属性
- **基本情報**: ファイル名、パス、サイズ、種類、日時
- **作品情報**: タイトル、原作者、作画者、よみがな
- **出版情報**: 巻数、ジャンル、出版社、発行日
- **管理情報**: タグ、AI処理済みフラグ

#### ファイル名解析
- 正規表現による自動解析
- カスタマイズ可能なパターン設定
- デフォルトパターン: `[一般コミック] [よみがな] [作者×作画] タイトル 第X巻`

### 3. AI 機能

#### Google Gemini API 連携
- 作品タイトルと作者名から詳細情報を取得
- バッチ処理による一括取得
- 同時処理数の制御（10-100リクエスト）
- エラーハンドリングとリトライ機能

#### 取得情報
- 発行日
- タグ情報
- AI処理済みフラグの管理

### 4. サムネイル機能（最適化版）

#### 高速サムネイル生成
- **並列処理**: 10並列での高速生成（CPUを適度に活用）
- **DB更新バッチ化**: 5秒間隔でまとめて更新（I/O負荷軽減）
- **解像度**: 600x400px（高品質維持）
- **形式**: JPEG（品質85%、設定可能）
- **保存先**: アプリフォルダ/thumbnails（権限エラー時はAPPDATAにフォールバック）
- **パフォーマンス**: 7万件を約40分で処理（従来比85%短縮）

#### 遅延読み込み最適化
- **非同期ファイルチェック**: UIスレッドをブロックしない
- **存在チェックキャッシュ**: 5分間のキャッシュで重複チェック削減
- **バッチUI更新**: 500ms間隔でまとめて描画更新
- **メモリキャッシュ**: LRU方式で200件キャッシュ（約40MB）
- **表示解像度**: 480px（DataGrid表示180px、ポップアップ600px対応）

#### ポップアップ機能
- **マウスオーバー**: 大きなサムネイル表示（最大600x400px）
- **表示遅延**: 即座表示、3秒自動非表示（設定可能）
- **画面端調整**: 自動位置調整で画面外表示を防止
- **高品質表示**: 元解像度でのクリアな表示

#### 自動削除機能
- **条件付きクリーンアップ**: 件数が1000件を超えた場合のみ実行
- **保持期間**: 30日（設定可能: 7-90日）
- **実行タイミング**: アプリ終了時 + 起動時チェック
- **手動クリーンアップ**: 設定に関係なく常時利用可能

### 5. 検索・フィルタ機能

#### リアルタイム検索
- 全属性を対象とした部分一致検索
- 入力と同時に結果を更新
- 大文字小文字を区別しない検索

#### フォルダフィルタ
- 特定フォルダのファイルのみ表示
- 「すべて表示」による フィルタ解除

### 6. UI・操作機能

#### DataGrid 機能
- 17列の詳細情報表示
- 列の表示/非表示切り替え
- 列幅とレイアウトの保存/復元
- ソート機能
- 複数選択対応

#### ダブルクリック動作
- **パス列**: エクスプローラーでフォルダを開く
- **ファイル名列**: 漫画ビューアで開く
- **編集可能列**: 編集モードに移行
- **その他**: デフォルトアプリで開く

#### キーボードショートカット
- **Ctrl+C**: 選択行の全データをコピー
- **Ctrl+Shift+C**: 選択列の値のみをコピー
- **Delete**: フォルダ削除（フォルダリスト）

### 7. 設定機能

#### 表示設定
- サムネイル表示の ON/OFF
- 列表示の個別制御
- ウィンドウ位置・サイズの保存

#### AI 設定
- Gemini API キーの設定
- 同時処理数の調整（10-100）

#### ビューア設定
- 外部漫画ビューアアプリの指定

#### ファイル名解析設定
- 正規表現パターンのカスタマイズ
- パターンのテスト機能
- デフォルトパターンへのリセット

#### メモリ設定
- 最大メモリ使用量（1-16GB）
- キャッシュサイズ（256MB-4GB）
